## Prompting Principles Applied (from AI_Prompting_Masterguide)

1. **Lead with the most important element** — subject first, then environment, then lighting
2. **Use positive framing** — describe what you want, never what you don't
3. **Think in layers** — Subject → Environment → Lighting → Style → Technical
4. **Use concrete nouns and vivid verbs** — no "beautiful," "amazing," "stunning"
5. **Specify the medium** — photographic terminology, lens type, film stock
6. **One hero element per shot** — the website mockup is the hero for thumbnails
7. **Specify aspect ratio** — 3:2 for thumbnails (1200×800), 1:1 for avatars
8. **Reference real-world materials** — screen glare, desk surface texture, ambient reflections

## Generation Workflow

### Recommended Pipeline (Nano Banana Pro via Higgsfield or Google AI Studio)

1. **Generate reference sheet first:** Create one project thumbnail, review, iterate until style is locked
2. **Use that as a style reference:** Feed the approved thumbnail as `--sref` (Midjourney) or reference image (Nano Banana) for all remaining thumbnails to maintain visual consistency
3. **Generate avatars as a batch:** Use consistent background and lighting instructions across all 9 avatars
4. **Generate portrait last:** This is the most important image — spend iteration budget here

### Midjourney V7/V8 Pipeline

1. Use `--v 7 --style raw` for all generations
2. Aspect ratios: `--ar 3:2` (thumbnails), `--ar 1:1` (avatars/favicon), `--ar 4:5` (portrait)
3. Generate 4 variations per prompt (`/imagine` produces 4 by default)
4. Upscale the best result with `U1`, `U2`, `u3`, or `u4`
5. If needed, vary the upscaled result with `Vary (Strong)` or `Vary (Subtle)`
6. For style consistency across thumbnails: generate Image 1 first, get its URL, then use `--sref [URL] --sw 300` on all remaining thumbnails

### Post-Processing

After generation:

1. **Resize:** Project thumbnails to 1200×800px, avatars to 400×400px
2. **Convert to WebP:** Use `cwebp -q 85 input.png output.webp` for optimal web performance
3. **Compress:** Ensure all thumbnails are <500KB
4. **Upload to Payload CMS:** Via admin panel at `/admin` → Media → Upload, or via API
5. **Link to projects:** Edit each project in Payload admin, attach the corresponding thumbnail
