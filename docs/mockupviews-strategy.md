## Overview

MockupViews is a free, browser-based mockup display tool — a direct alternative to shots.so. It allows designers and developers to showcase website screenshots inside realistic device frames (desktop browsers, laptops, tablets, phones) with customisable backgrounds, colours, and styling. No account required, no watermarks, no limitations.

For Edmond's practice, MockupViews serves as:

1. **Portfolio presentation** — Displaying completed client projects in polished device mockups for the personal website portfolio
2. **Proposal and pitch visuals** — Including professional mockups in proposals to show prospective clients what their completed project could look like
3. **Social media content** — Creating visually compelling project showcase images for LinkedIn, X, Instagram, and Threads
4. **Case study imagery** — Professional before/after or final-state mockups for written case studies and project documentation

## By Service Category

### 1. Web Design & Development

#### Portfolio Project Display
- **Use case:** After completing a client project, take a screenshot of the live site → upload to MockupViews → select appropriate device frame (desktop for full site, phone for mobile view) → customise background to match the project's brand colours → download → add to portfolio gallery on personal website
- **Value:** Professional portfolio presentation without paying for shots.so, Placeit, or other premium mockup tools. The output is indistinguishable from paid alternatives
- **Workflow:** Screenshot (Full Page Screen Capture extension) → MockupViews → device frame → download → upload to portfolio

#### Proposal Visuals
- **Use case:** During a proposal or discovery call, show the prospective client a mockup of what their current site could look like redesigned. Use a wireframe or design mockup → MockupViews → present in a realistic device context
- **Value:** Prospects visualise the outcome more concretely when they see it inside a device frame rather than as a flat image. This increases proposal close rates

#### Before/After Comparisons
- **Use case:** For redesign projects, create a before/after pair — old site screenshot in a muted/greyscale frame, new site in a vibrant frame with matching brand colours → place side by side
- **Value:** Dramatic visual proof of capability. "Here's what we started with, here's what we delivered" is the most persuasive portfolio format

## Quick-Win Implementations

### Priority 1: Standard Portfolio Mockup Workflow (15 min per project)
```
1. Open completed site in browser
2. Take full-page screenshot (use browser dev tools or extension)
3. Go to mockupviews.com
4. Upload screenshot
5. Select device frame (desktop browser for full site, phone for mobile)
6. Adjust background colour to match project's brand palette
7. Fine-tune rotation, shadow, and padding
8. Download high-resolution PNG
9. Upload to personal website portfolio
```

### Priority 2: Responsive Design Mockup Set (30 min per project)
Create a 3-image set for each portfolio project:
- **Desktop view:** Laptop or desktop browser frame — shows the full layout
- **Tablet view:** iPad frame — shows the intermediate breakpoint
- **Mobile view:** Phone frame — shows the mobile-first design

Upload all three as a responsive design gallery on the portfolio.

### Priority 3: Proposal Template Library (1 hour)
Create pre-styled MockupViews templates for common proposal scenarios:
- **Landing page mockup:** Single-page layout in desktop frame
- **E-commerce mockup:** Product page in desktop + phone frames
- **Booking platform mockup:** Calendar/booking UI in desktop frame
- **Portfolio mockup:** Gallery layout in desktop frame

Save the settings (background colour, frame type, rotation) as a reference document so you can recreate them quickly for each proposal.

## Risks & Considerations

1. **Browser dependency:** MockupViews runs entirely in the browser. If the site goes down or changes its model, the tool becomes unavailable. **Mitigation:** Download mockups immediately after creation — don't rely on re-accessing them later
2. **Limited frame selection:** Compared to Placeit (10,000+ templates), MockupViews has a smaller selection of device frames. For most web design purposes, the available frames (desktop, laptop, tablet, phone) are sufficient
3. **No batch processing:** Each screenshot must be uploaded and styled individually. For large portfolios (10+ projects), this can be time-consuming
4. **Image quality:** Output quality depends on the input screenshot resolution. Always use full-page, high-resolution screenshots — not cropped or resized images
5. **No video mockups:** MockupViews handles static images only. For video or animated mockups, consider Google Flow (Veo 3.1) or manual screen recording inside device frame overlays
